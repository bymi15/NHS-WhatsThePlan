Use Cases

Use Cases are a list of actions between actors and the system to achieve some goal.


ID:

UC1

Requirement ID:

F1

Description:

The user edits their demographic details.

Actors:

User

Preconditions:

The user is logged into the app.

Main Flow:

1. The use case starts when the User accesses their demographic details by tapping the ‘My Profile’ icon on the main page.

2. The app displays the User’s profile which includes their demographic information: full name, date of birth, gender, nationality, marital status and also GP name and surgery.

3. The User selects the menu icon on the top bar of the app.

4. The menu list collapses, showing the ‘Edit’ button in the list.

5. The User taps the ‘Edit’ button

6. The app displays the profile information in textboxes.

7. The User makes some changes to their information and selects the ‘Save changes’ button.

Post Conditions:

The app saves the changes to the database and updates the contents of the Profile page.

Alternate Flows:

 

 

ID:

UC2

Requirement ID:

F2, F3

Description:

The user adds a medicine to their medications list

Actors:

User

Preconditions:

The user is logged into the app.

Main Flow:

1. The use case starts when the User accesses their medications by tapping the ‘Medication Manager’ icon on the main page.

2. The app displays a list of the User’s medications.

3. The User selects the + icon on the top bar of the app.

4. The app displays a form with a dropdown textbox, which allows the User to search for known medications or select one using the dropdown menu, textboxes for the dosage, frequency and occurrences of the medication, and a calendar input text field for the start date and end date.

5. The User fills in the form and selects the ‘Add’ button.

 

Post Conditions:

The app adds a new record to the Medicines table in the database, containing the information inputted by the user and updates the contents of the Medicine Manager page, displaying it in the list.

Alternate Flows:
