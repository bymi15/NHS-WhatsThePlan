Unit and Integration Testing

Unit testing is responsible for testing the individual components of the app to ensure that they work as expected. The unit testing will be performed on both the backend (Firebase) and the frontend (Ionic) of the app.
Unit testing will be performed after any change is made to the code, to ensure that the changes do not break the app.

Integration testing is where we will write end-to-end tests which perform a sequence of a use case and will ensure that the app is functioning properly as a whole â€“ each component is working properly together as a connected system.
These tests will usually take longer to perform and will be done after unit testing.


Jasmine is a behaviour driven development framework for testing JavaScript code so it is suitable for testing Ionic apps, as Ionic is built with AngularJS, a JavaScript framework.


Karma is a tool built by the Angular team for running unit tests on a browser, so Karma can be used to run the tests written in Jasmine.



Automated Testing

Both the unit tests and integration tests will be automated as they will need to be performed frequently throughout the development of the system. For automating the tests, we will be using Grunt.



Grunt is a task runner which allows us to build and perform tests automatically.
It also allows us to install all the testing frameworks such as Jasmine and Karma in a single command and then execute the tests in a project during the build process, so all the tests are run automatically after the project code is built.



User Acceptance Testing (UAT)

UAT is the final stage of testing that occurs after all the previous tests: unit tests and integrations tests have passed. For our project, we will be using alpha-beta testing.

Alpha testing will take place in the development environment and will be performed by us, the developers.

Beta testing will take place in an external environment. We will be communicating closely with our client to perform beta testing with our client after every major iteration of the development process and ensure that our client is satisfied with the product.

Feedback from the alpha-beta testing will be used to make improvements to the product.
